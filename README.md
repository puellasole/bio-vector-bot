# 🧬 BioVector

ИИ-репетитор по биологии в Telegram для подготовки к ЕГЭ. Бот выдаёт задания из базы, принимает ответ ученика и отправляет его на проверку нейросети DeepSeek, которая возвращает детальный разбор ошибок, рекомендации и оценку по критериям ЕГЭ.

## 🚀 Возможности

- **Задания первой части** — установление соответствия, множественный выбор, последовательности (типы 6–20)
- **Задания второй части** — развёрнутые ответы с проверкой по ключевым элементам (типы 22–28)
- **ИИ-проверка через DeepSeek** — нейросеть анализирует ответ, находит ошибки, объясняет биологические неточности и выставляет баллы
- **Автоматическая проверка** — для заданий первой части с эталонным ответом проверка происходит мгновенно без запроса к нейросети
- **Система наград (XP)** — баллы за правильные ответы, статусы от «Начинающего» до «Мастера биологии»
- **Отправка изображений** — к заданиям прикрепляются картинки из resources
- **Inline-клавиатура** — актуальные команды под рукой после каждого действия
- **Сброс состояния** — команда `/quit` позволяет прервать текущее задание и выбрать другое

## 📋 Команды бота

| Команда | Описание |
|---|---|
| `/start` | Приветствие и выбор части |
| `/first` | Задания первой части |
| `/second` | Задания второй части |
| `/stats` | Статистика наград (XP) |
| `/help` | Справка по использованию |
| `/quit` | Сброс текущего задания |

## 🔄 Как это работает

### Первая часть (тестовая)
1. Ученик выбирает `/first` и номер задания (например, `/6001`)
2. Бот загружает задание из БД, отправляет текст и картинку
3. Ученик вводит ответ (последовательность цифр)
4. Если у задания есть эталонный ответ — бот сверяет мгновенно и начисляет XP
5. Если эталона нет — запрос уходит в DeepSeek для проверки

### Вторая часть (развёрнутая)
1. Ученик выбирает `/second` и номер задания (например, `/2201`)
2. Бот загружает задание и отправляет ученику
3. Ученик пишет развёрнутый ответ
4. Ответ отправляется в DeepSeek с промптом, содержащим критерии оценки
5. Нейросеть возвращает: пошаговый анализ по ключам, биологические ошибки, итоговый балл, рекомендации и образец правильного ответа
6. Бот извлекает балл из ответа нейросети и начисляет XP

## 🧠 Prompt Engineering

Для каждого типа заданий разработаны отдельные системные промпты:

| Тип задания | Логика проверки |
|---|---|
| `matching` | Побуквенное сравнение с эталонной последовательностью |
| `multiple_choice` | Поиск пропущенных и лишних вариантов с биологическим обоснованием |
| `sequence_order` | Позиционное сравнение с анализом нарушенной логической цепочки |
| `second_section` | Оценка по ключевым элементам со штрафами за биологические ошибки |

Каждый промпт задаёт:
- Роль эксперта ЕГЭ
- Правила начисления и снятия баллов
- Строгий формат ответа (без разметки)
- Требование объяснять биологический смысл ошибок

## 🗄 Схема базы данных

### `tasks`
Хранит банк заданий: номер, текст вопроса, правильный ответ, ключи для проверки, тип задания, путь к картинке.

### `user`
Хранит данные пользователя: chat_id и количество XP.

## 🏆 Система наград

| XP | Статус |
|---|---|
| 0–9 | 🌱 Начинающий |
| 10–29 | 📚 Ученик |
| 30–59 | 🎓 Знаток |
| 60+ | 🏅 Мастер биологии |

- Первая часть: +10 XP за правильный ответ
- Вторая часть: +5 XP за каждый балл

## ⚙️ Конфигурация

В `application.properties` необходимо указать:

```properties
bot.token=your_telegram_bot_token
deepseek.url=https://api.deepseek.com/v1/chat/completions
deepseek.api.key=your_deepseek_api_key
```

## 🚧 Особенности

Таймауты HTTP-клиента увеличены до 120 секунд для ожидания генерации ответа нейросетью

Состояние сессии пользователя хранится в ConcurrentHashMap (состояние, сценарий, активное задание)

Изображения заданий хранятся в resources/images/ и отправляются через SendPhoto

При смене задания или выходе /quit состояние сессии полностью очищается
